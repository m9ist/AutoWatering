# Автополив

Система автополива до 16 комнатных растений: Mega2560 управляет железом,
ESP8266 связывает её с внешним миром через MQTT-брокер на домашнем серваке.

## Language

### Устройства и сервисы

**Mega**:
Arduino Mega2560, управляет железом (насос, клапаны, датчики, дисплей, SD).
_Avoid_: ардуина (двусмысленно — ESP тоже Arduino-прошивка)

**ESP**:
ESP8266, headless-шлюз между Mega (UART) и брокером (MQTT). Своего
пользовательского интерфейса не имеет.

**aw-server**:
Серверный сервис на deb: телеграм-бот автополива, мост логов в Loki,
приёмник телеметрии. Код — в `server/` этой репы, деплой — в репе homelab.
_Avoid_: бот (это только одна из его ролей), мост

**Брокер**:
Локальный Mosquitto на серваке deb — единственная точка подключения ESP
и общая шина для будущих домашних интеграций.
_Avoid_: Dealgate (внешний брокер, ESP к нему не подключается — см. ADR-0001)

### Потоки данных

**Команда**:
JSON с ключом `c`, адресованный Mega. Формат един на всём пути
Telegram → aw-server → MQTT (`aw/cmd`) → ESP → UART; ESP валидирует
и форвардит, не переформатируя.

**Стейт**:
Снимок состояния системы (растения, климат, флаги), который Mega
периодически шлёт наверх. Последний стейт лежит retained в `aw/state`.

**Событие**:
Сообщение от ESP/Mega, предназначенное человеку в Telegram (подтверждения,
отказы Команд, тревоги). Публикуется в `aw/event`, доставляет aw-server.

**Лог-поток**:
Строки логов ESP, уходящие в `aw/log/esp` и далее в Loki. Дублируются
по старому пути UART → Mega → SD (офлайн-фолбэк).

### Полив

**Растение**:
Позиция 0..15: клапан + датчик влажности + дневная норма (мл).

**Дневной полив**:
Полив всех растений по их нормам одной командой (`/daily`); запускается
вручную, автозапуск по расписанию — в планах.

**Норма**:
Настроенный дневной объём воды растения, мл (`dailyAmountMl`).
_Avoid_: конфиг растения

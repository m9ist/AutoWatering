## Методика тестирования п.28 (SPI_DRIVER_SELECT через build_flags)

Проблема прошлых попыток скорее всего в том, что руками правленный
`SdFatConfig.h` уже лежал в `.pio/libdeps` и перебивал флаг, либо флаг
добавлялся не в то окружение. Порядок проверки:

1. **Чистая библиотека**: удалить `.pio/libdeps/megaatmega2560/SdFat`
   (или весь `.pio/libdeps/megaatmega2560`), чтобы PlatformIO скачал
   нетронутый SdFat — иначе тест ничего не докажет.
2. В `[env:megaatmega2560]` добавить `build_flags = -D SPI_DRIVER_SELECT=2`.
3. **Проверка на компиляции**: `SdFatConfig.h` оборачивает дефолт в
   `#ifndef SPI_DRIVER_SELECT` — значит внешний `-D` должен подхватиться.
   Убедиться: `pio run -e megaatmega2560 -v | grep SPI_DRIVER_SELECT` —
   флаг присутствует в командах компиляции; сборка без ошибок про
   `SoftSpiDriver` (класс существует только при SPI_DRIVER_SELECT=2,
   т.е. если бы флаг не подхватился — `AwLogging.h` просто не собрался бы).
4. **Проверка на железе**: прошить, смотреть Serial на boot:
   должно быть `SD is inited.` (а не `sd.initErrorPrint`). Дальше дождаться
   пары записей и проверить LOG.txt на карте с компа.
5. Если ок — убрать упоминание ручной правки из readme (шаг 1 в
   «Обязательные шаги перед сборкой»).

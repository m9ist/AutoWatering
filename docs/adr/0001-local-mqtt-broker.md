# ESP говорит только с локальным MQTT-брокером; интеграции — на стороне сервака

Дата: 2026-07-20. Статус: accepted (заменяет план прямого подключения к Dealgate
из `tasks/2026-07-04_mqtt_dealgate_implementation_plan.md`).

ESP8266 держит единственное внешнее соединение — MQTT к локальному Mosquitto на
домашнем серваке (deb, стек `homelab/stacks/mqtt`, порт 1883 в LAN, user/pass,
без TLS). Все интеграции наружу (Telegram, Loki, в будущем Алиса через
mosquitto-bridge в Dealgate или навык «Маша» из ТЗ «Джарвис») живут на серваке.

Почему так, а не по плану 2026-07-04 (ESP → mqtt.dealgate.ru напрямую):

- Логи в Loki и телеграм-релей всё равно требуют локального брокера — Dealgate
  (чужой публичный брокер, глобальное пространство топиков, лимит 300 pub/мин)
  для них непригоден. Два брокера с ESP8266 — лишние RAM и точки отказа.
- Telegram на ESP на момент решения не работал (блокировки/VPN); на серваке
  VPN-слой с failover уже решает эту проблему для серверного бота.
- Прошивка перестаёт зависеть от выбора способа интеграции с Алисой — решение
  откладывается без блокировки остальной работы.

Следствия: из прошивки уходят UniversalTelegramBot, весь TLS (BearSSL),
SettingsGyver (веб-морда) и код Yandex IoT; ESP становится headless-устройством
(OTA + UART + MQTT). Наблюдение — Grafana/Loki на серваке, управление — бот
aw-server (код в `server/` этой репы, деплой — в homelab).

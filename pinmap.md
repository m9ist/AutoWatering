## 74HC595 — 3 шт. в каскаде

Управление: DS=D34, SHCP=D38, STCP=D36. OE и MR — на VCC/GND статически.

В каждом регистре используется **только Q1..Q7** (7 выходов из 8). 3 × 7 = 21 канал, на клапаны идёт 16, остальные пустуют. Логика — в `Pomp.h::patchValveId`:

```c
if (id < 7)  return id + 1;   // valves 0..6  → register1.Q1..Q7
if (id < 14) return id + 2;   // valves 7..13 → register2.Q1..Q7
else         return id + 3;   // valves 14..15 → register3.Q1..Q2
```

С каждого регистра Q1..Q7 → IN1..IN7 соответствующего ULN2003 → клапан + флайбэк-диод (диод уже в ULN2003, отдельные не нужны).

## ULN2003 — 3 шт.

Стоят сразу после 74HC595, по одной на регистр. Питание (COM, pin 9) — от шины **5V клапанов** (отдельная от 5V логики).

**Важная особенность подключения клапанов:** ULN2003 — это сборка open-collector NPN, она **замыкает массу**, а не подаёт плюс. Поэтому:

- **+5V (шина клапанов)** подаётся на **все 16 клапанов** одной общей шиной (плюсовая ножка каждого клапана).
- **Минусовая ножка** каждого клапана уходит на индивидуальный выход OUT драйвера ULN2003.
- COM-пин ULN2003 (pin 9) тоже подключён к **+5V шины клапанов** — для встроенных freewheel-диодов.

**Что это даёт для кабельного хозяйства / будущей PCB-перепланировки:**
один толстый провод +5V (общий) + 16 сигнальных в плоском шлейфе. Хорошо для будущей архитектуры с отдельной саб-платой клапанов и USB-кабелем между платами.

## Силовая часть (as-is)

```
USB-C PD-trigger (модуль с DIP-switch, 5/10/15/20V) → 20V
   │
   ├── LM2596S #1 → 12V → MX1508 → 2 насоса R385
   │
   └── LM2596S #2 → 5V (общая шина) → клапаны через ULN2003
                                    + Mega (на пин 5V), регистры, мультиплексоры,
                                      датчики, дисплей через 3.3V LDO самой Mega
```

**Важно:** в текущей as-is одна общая шина 5V на всё. Разделение на «5V клапанов» и «5V логики» — в [backlog.md](backlog.md), на рефакторинг.

3.3V для TFT берётся с самой Mega; SPI-сигналы (SCK/MOSI/DC/RES) проходят через резистивные делители 5V→3.3V (R 2.2k + R 3.3k).

## Подключение MX1508 (помпы R385)

Реверс не нужен — помпы крутятся только в одну сторону. PWM на одном входе пары, второй вход — на GND:

| Канал MX1508 | Подключение |
|---|---|
| Помпа основная | IN1 = D6 (PWM), IN2 = GND. Помпа между OUT1 и OUT2 (или OUT2 и GND, если плата так разведена). |
| Помпа запасная | IN3 = D5 (PWM), IN4 = GND. Аналогично. |

Питание VCC драйвера = **12V** (с LM2596S #1). Логическая сторона MX1508 (VCC IO) — обычно объединена с VCC-силовой, нужно проверить по конкретному модулю.

## Питание Mega

**Внешний 5V с LM2596S #3 идёт прямо на пин `5V` Mega-модуля** (минуя встроенный регулятор Mega). Та же шина уходит и на нижнюю плату (питание регистров, мультиплексоров, ULN2003 — последний только в COM, не на VCC).

USB-разъём Mega остаётся для прошивки/отладки, но в постоянной работе питание идёт по 5V-пину.

## Архитектура датчиков с pogo

Датчики (расходомер, уровень воды) — внешние устройства с 3-проводным кабелем. На конце кабеля припаян pogo, на плате — ответный pogo. Так датчик легко отстёгивается для обслуживания.

Дальше с pogo'a на плате:
- **V+ → шина 5V** (на той же плате)
- **GND → шина GND** (на той же плате)
- **Сигнал → межплатный USB → Mega** (на верхней плате — D2 для расходомера, A1 для уровня воды)

То есть pogo физически стоит на нижней плате, а сигнал везётся на верхнюю через межплатный кабель.

## Межплатный коннектор

На обеих платах стоит **USB 3.0 (тип A или B)**. Между платами — обычный USB 3.0-кабель, по 9 жил которого передаются: 5V, GND и нужные сигналы между нижней и верхней платой.

Маппинг конкретных сигналов на пины USB-разъёма зафиксируем при разводке.
